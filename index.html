[USACO] Section 2.3 Cow Pedigrees

經典DP題:
給定 數N 樹高K
請問有幾種degree為0或2的二元樹?

首先
當然可以很輕易地想到DP狀態轉移是 : 左子樹方法數乘以右子樹方法數
也就是以定義狀態為f(x, y) : 用x個點, 高度y, 能弄出幾種
那麼在狀態轉移時

f(x, y) = sum(f(k, h1) * f(x - k - 1, h2))

其中 1 <= k <= x-1, 1 <= h1, h2 <= y-1

k為左子樹的點數
x-k-1為柚子樹的點數

h1為左子樹的高度
h2為柚子樹的高度

赫然發現
問題變得更複雜了!!
居然要枚舉三個維度 : 左/右子樹點數, 左子樹高度, 柚子樹高度
才能完成狀態轉移

「DP心得之一 : 狀態的選擇是很重要滴!!」

把DP狀態改成 : f(x, y)為用x個點, 湊出「最大高度為y」的方法數
狀態轉移為f(x, y) = sum(f(k, y-1) * f(x-k-1, y-1)), 1 <= k <= x-1

則題目要求的剛好高度為y的方法數就是f(x, y) - f(x, y-1)

用比較複雜的問題來解原問題, 反而比較比較輕鬆
其實大學何錦文教授有講過類似的概念
不過那時也沒在聽就是了
嘻嘻

其實這題還有後續

「DP心得之二 : 狀態轉移是很重要滴!!」

因為左右子樹長相是對稱的
所以還能省一半轉移的時間

但注意兩邊點數一樣話就是同一種

最後補充一下
所謂degree為0或2其實隱含的被描述在
狀態轉移時的k的範圍是 1 ~ x-2, 不可以有一邊的點數是0

也就是說這題還能再推廣成「真正的」二元樹的方法數

[USACO] Section 2.3 Longest Prefix

簡單DP題

令f(x, y)代表用前x個primitives, 能否湊出長度y的prefix
則f(x, y) = f(x-1, y) | f(x-1, y-len(primitives[x]))

f(x-1, y) 代表不用第x個primitives
f(x-1, y-len(primitives[x])) 代表用了第x個primitives

x最大是200
y最大是200000
直接開200 * 200000的陣列會把stack撐爆

所以非得滾動數組不可

[USACO] Section 2.2 Party Lamps

不簡單題

第一眼 會以為又是典型的BFS題
但看到Button可以按10000次就該知道事有蹊蹺

觀察1:
每個Button都是按兩次等於沒按
所以其實從頭到尾就只有2^4種可能
而這2^4種可能之中還有重複的, 但我就沒管它了, 直接bool擋掉

觀察2:
Button2是顛倒奇數
Button3是顛倒偶數
Button4是顛倒1, 4, 7, 11, …, 3N+1
所以真正要維護的狀態只有6個bit

觀察3:
只要C大於「要按的Button個數」並且奇偶一樣, 才可以到達想要的狀態

最後要注意 答案要求的是「字典序的大小」
跟維護6個bit的「數字的大小」是不一樣的
所以我把string拿去std::sort

看似簡單的一題
但繞了好幾個彎
程式解題競賽果然是給年輕人玩的東西啊…

Note:
USACO的測資看起來滿弱的
不確定到底是否正解XD

[USACO] Section 2.2 Subset Sums

經典DP題

arr[i]代表第i個數字
令f(x,y)代表用前x個數字 湊出總和y的方法數
則f(x, y) = f(x-1, y) + f(x-1, y-arr[i])

f(x-1, y) 代表不用第x個數字, 也能湊出y的方法數
f(x-1, y-arr[i]) 代表用到第x個數字

因為本題是問湊出總和一半的方法數
所以得出來的f(N, sum/2)要再除2

[USACO] Section 2.1 Sorting a Three-Valued Sequence

不簡單題

最直觀的做法就是O(n^2)的貪心
優先換互相在正確位置的配對
真的不行才換不成對的配對

O(n)的作法
一樣是先換互相在正確位置的配對
然後運用一個觀察: 如果不成對的配對交換, 那可以保證再換一次就一定可以到正確位置
例如: 把在「2區間」的1換到「1區間」, 如果換回「2區間」的是3, 而不是1
那一定剛好會有一個在「3區間」的2等著你

苦思良久
沒有自己想出來 可惡QAQ
google搜了三頁
國人的解題報告作法都超醜XDD
總算在stack overflow看到這個順眼又直觀的解法

推廣:
如果是固定k distinct value sequence最少交換幾次可以排序好?
